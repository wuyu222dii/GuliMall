This is a brand new way of usage. It should be said that this is the mainstream in the future and is also the currently recommended usage of the platform. For related introduction, please refer to this article This usage actually makes asvgThe set has the following characteristics compared with the other two:

  • Multi-color icons are supported and are no longer limited by single color.
  • With some tricks, support like fonts, throughfont-size,colorto adjust the style.
  • Poor compatibility, support ie9+,and modern browsers.
  • Browser renderingsvgThe performance is average, not as good aspng.

The usage steps are as follows:

Step 1: Import the code generated under the projectsymbolCode:

<script src="./iconfont.js"></script>

Step 2: Join GeneralcssCode (just introduce it once):

<style type="text/css">
.icon {
   width: 1em; height: 1em;
   vertical-align: -0.15em;
   fill: currentColor;
   overflow: hidden;
}
</style>

Step 3: Select the corresponding icon and get the class name, and apply it to the page:

<svg class="icon" aria-hidden="true">
  <use xlink:href="#icon-xxx"></use>
</svg>
